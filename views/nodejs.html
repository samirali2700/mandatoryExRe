NodeJs

Nodejs is a javascript enviromnment, javascript used to only be available on browsers (frontend),because javascript is such an easy an usefull language nodejs were developed to run javascript outside browsers, meaning javascript could be used outside browser frontend.

Nodejs uses the same engine as Chrome Browser an engine called V8 Engine

Besides coding javascript outside browsers, nodejs via npm (node package management) opens a world filled with third party packages that are easy integrated and used with ones application.
An example of this is Web Framework Express which is installed through npm

Npm

Nodejs can be downloaded and installed from browser or from third party package manager. When nodejs is installed, npm will be available globally. Npm can be used to install packages to a project that are dependent on some packages, these packages are defined in package.json

This is dependencies object inside package.json. Any Key/Property here is a project dependent package, in this case express and fs are packages used in the project and the application will not function properly without them


Npm can install all packages that are listed inside package.json, without the need to install them one by one

If node_modules is none existing, and package.json is listed with different packages to install them all

npm install or npm i

To See installed packages in terminal

npm list

Reminder

If extra steps are not made to exclude node_modules folder from repository, a big chunk of unnecessary space will be filled by node_modules folder

to prevent this, either before first commit, add '.gitignore' file to the root project folder or add it when creating the repository, then add the folder name to the file

Then from then on, every commit will exclude the node_modules folder, and each and everyone fetching this repository will have to install them locally
